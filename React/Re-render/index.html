Re-render

什麼時候會造成 re-render​

  • Parent Component 的 state 改變，React 會調用 Parent Component 的 render 方法，來產生新的 Virtual DOM，和上一個 Virtual DOM 比較後，找到不同的地方，反應到 真實的DOM 上面，這時候底下所有的 child component 都會被重新 render。
  • 但如果 child component 和被改變的 state 無關，我們不希望他也被重新 render，造成效能上的浪費。

怎麼避免元件不必要的 re-render​

  • 我們希望只有元件本身的 props 改變時，元件才會重新 render。
  • 這邊會用到 React 提供的兩個 Hooks，memo 和 useMemo。

memo​

只有 props 改變才重新 render component

const MyComponent = React.memo(function MyComponent(props) {
  /* render using props */
});

先看一下 React 官方的範例和說明:

/*
If your component renders the same result given the same props, you can wrap it in a call to React.memo for a performance boost in some cases by memoizing the result. This means that React will skip rendering the component, and reuse the last rendered result.

如果 render 的時候你的 component 接收的 props 和原本的一樣，你可以用 React.memo 包住 component 透過記住原本的結果來提高效能。React 會跳過這個 component，直接使用他上一次(原本) render 的結果。
*/

/*
React.memo only checks for prop changes. If your function component wrapped in React.memo has a useState, useReducer or useContext Hook in its implementation, it will still rerender when state or context change.

React.memo 只有在 props 改變的時候會重新執行，如果你的 component 裡面有用到 useState, useReducer, useContext 等 Hook，當這些狀態改變的時候 component 還是會重新 render。
*/

/*
By default it will only shallowly compare complex objects in the props object. If you want control over the comparison, you can also provide a custom comparison function as the second argument.

memo 預設是用淺拷貝來比較 object 的 props，如果你想要對這部分調整，可以把自定義比較的 function 當作第二個參數
*/

useMemo​

只有 dependencies 改變才重新執行 function

// 傳入一個 function 和 dependency array，function return 一個 value
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);

一樣先看 React 官方的範例和說明:

/*

回傳一個被記住的值
Returns a memoized value.

在裡面放入一個 function 和 dependencies，useMemo 只會在 dependencies 的值改變時對被記住的值做重新運算，避免每次 render 時 function 都重複進行大量運算。
Pass a “create” function and an array of dependencies. useMemo will only recompute the memoized value when one of the dependencies has changed. This optimization helps to avoid expensive calculations on every render.

記得，在 useMemo 裡面的 function 會在 render 的時候執行，不要在裡面做可能造成非預期行為的事，像 side effects 請使用 useEffect。
Remember that the function passed to useMemo runs during rendering. Don’t do anything there that you wouldn’t normally do while rendering. For example, side effects belong in useEffect, not useMemo.

如果沒有 dependencies array，每次 render 都會重新計算 value
If no array is provided, a new value will be computed on every render.

你可能會把 useMemo 當作效能優化的手段，但使用 useMemo 不保證是"優化"
You may rely on useMemo as a performance optimization, not as a semantic guarantee.

*/

實際使用 memo 和 useMemo​

CodeSandBox - memo, useMemo

import { useState, memo, useMemo } from "react";

export default function App() {
  const [count, setCount] = useState(0);
  const [mode, setMode] = useState("dark");

  const handleCount = () => {
    setCount((prev) => prev + 1); // render 次數 +1
  };
  const handleMode = () => {
    setCount((prev) => prev + 1); // render 次數 +1
    setMode(mode === "dark" ? "light" : "dark"); // 更新 mode
  };

  const style =
    mode === "dark"
      ? { name: "Child", mode: "light" }
      : { name: "Child", mode: "dark" };

  // ** 只有 mode 改變時，useMemo 內的 function 才會重新執行 **
  const memoStyle = useMemo(() => {
    return mode === "dark"
      ? { name: "memoChild", mode: "light" }
      : { name: "memoChild", mode: "dark" };
  }, [mode]);

  // 上面的 button 計算 render 次數，另一個 button 切換 mode
  // 點上面的 button 不會讓 MemoChild 重新 render，點下面的才會
  return (
    <div className="App">
      <h2>重新 render 次數: {count}</h2>
      <button onClick={handleCount}>add render times</button>
      <button onClick={handleMode}>change mode</button>
      // ** **
      <Child style={style} />
      <MemoChild style={memoStyle} />
    </div>
  );
}

// ** memo 包住的 component，只有 props 改變才會重新 render **
const MemoChild = memo(PropsNotChange);
function PropsNotChange(props) {
  console.log("MemoChild re-render");
  return (
    <div>
      {props.style.name} mode: {props.style.mode}
    </div>
  );
}

function Child(props) {
  console.log("child re-render");
  return (
    <div>
      {props.style.name} mode:{props.style.mode}
    </div>
  );
}
